Zero Duplication

Define class names and their styles together in one place. The old approach required defining class names twice - once in classes and again in styles.

✨ View New Unified Styles API 
// 🎉 NEW: Unified Styles (No Duplication!)
defineComponent("smart-counter", {
  props: (attrs) => ({
    initialCount: parseInt(attrs.initialCount || "0"),
    step: parseInt(attrs.step || "1")
  }),
  styles: {
    // ✨ Class names extracted automatically from selectors!
    container: `.counter { display: flex; gap: 0.5rem; border: 2px solid #007bff; }`,
    button: `.counter-btn { background: #007bff; color: white; padding: 0.5rem; }`,
    buttonHover: `.counter-btn:hover { background: #0056b3; }`,
    display: `.count-display { font-size: 1.5rem; font-weight: bold; }`
  },
  render: ({ initialCount, step }, api, classes) => (
    <div class={classes.container}>
      <button class={classes.button}>-{step}</button>
      <span class={classes.display}>{initialCount}</span>
      <button class={classes.button}>+{step}</button>
    </div>
  )
});
📚 Compare: Old vs New Approach

❌ Old Way (Duplication)

classes: {
  container: "counter",        // ← Define class name here
  button: "counter-btn"        // ← And here
},
styles: \`
  .counter { display: flex; }     // ← Then repeat in CSS
  .counter-btn { padding: 1rem; } // ← More repetition
\`

✅ New Way (Unified)

styles: {
  container: \`.counter { display: flex; }\`,        // ← One place!
  button: \`.counter-btn { padding: 1rem; }\`       // ← Clean & DRY
}
