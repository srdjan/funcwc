defineComponent
Core API for creating components with modern function-style props and CSS-only format.
defineComponent("component-name", {
  render: ({ /* function-style props */ }) => JSX,
  styles: { /* CSS-only format */ },
  api: { /* HTMX endpoints */ }
});
‚úÖ Best Practice
Use function-style props with smart type helpers: string(), number(), boolean() for zero duplication.

defineReactiveComponent
Enhanced component API with hybrid reactivity system for complex state management.
defineReactiveComponent("reactive-name", {
  cssReactions: { /* CSS property reactivity */ },
  stateSubscriptions: { /* Pub/sub state */ },
  eventListeners: { /* DOM events */ },
  onMount: "/* initialization code */"
});
‚ö†Ô∏è Advanced Feature
Reactive components require the StateManager to be initialized. Include the StateManager script in your HTML.

üîç Troubleshooting

Common issues and their solutions when working with funcwc.
‚ùå
Component Not Rendering
Component tags appear as text instead of rendered components.
üîß Solution
  • Ensure component is imported in main.ts
  • Check JSX pragma: /** @jsx h */
  • Verify component name matches exactly
  • Import h function: import { h } from "funcwc"
üîÑ
State Not Updating
StateManager subscriptions not triggering updates.
üîß Solution
  • Include StateManager script in HTML
  • Check subscription topic names match exactly
  • Ensure publish() is called correctly
  • Verify component uses defineReactiveComponent
üé®
Styles Not Applied
CSS-only format styles not showing up.
üîß Solution
  • Use CSS properties only, no selectors
  • Check class names are auto-generated correctly
  • Verify classes! is passed to render
  • Ensure proper CSS syntax in template literals
üîó
HTMX Not Working
API endpoints not responding to HTMX requests.
üîß Solution
  • Include HTMX script and json-enc extension
  • Add hx-ext="json-enc" to body
  • Check API route registration in server
  • Verify HTTP method matches (GET, POST, PATCH, DELETE)
